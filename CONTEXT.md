# Project Context

This file captures the working context behind the extension so another agent or maintainer can continue without needing the original chat history.

## Goal

Build a PopClip extension that rewrites selected text using existing logged-in CLI tools instead of requiring API keys inside PopClip.

Primary use case:

- Select messy Turkish text anywhere on macOS.
- Run a PopClip action.
- Replace the selected text with a cleaner rewrite.
- Choose between providers such as Ollama, Codex, Claude, Gemini, and OpenCode when needed.

## Important Product Decisions

PopClip does not currently expose a public extension API for nested submenus, right-opening folders, or action dropdowns inside the PopClip bar.

What is supported:

- Multiple actions in one extension.
- Dynamic actions generated by JavaScript modules.
- Settings dropdowns through `multiple` options.
- Modifier-key behavior.
- PopClip's global overflow/expand UI when too many actions exist.

What is not supported:

- Extension-controlled submenu/folder/dropdown inside the PopClip action bar.
- A single action that opens native PopClip child actions.

- https://forum.popclip.app/t/how-to-build-nested-drill-down-options/1864

## Current Recommended Workflow

Public/simple version:

- Keep one PopClip icon.
- Choose provider/preset/model in PopClip settings.
- Use it for one chosen default workflow.

Recommended next local/power-user version:

- Keep one PopClip icon.
- On click, show a native macOS picker/list.
- Let the user choose combinations such as:
  - `Codex - Duzelt`
  - `Codex - Chat`
  - `Codex - Mail`
  - `OpenCode - Duzelt`
  - `Claude - Custom`
- Then run the selected provider/preset.

This is not a real PopClip submenu, but it is the closest practical workflow without cluttering the PopClip bar.

This picker workflow is now available through provider value `picker`.

There is also a faster one-icon workflow using PopClip's `modifier flags` script variable:

- No modifier: configured provider/preset.
- Option: Ollama / Chat.
- Shift: Ollama / Mail.
- Control: Ollama / Müşteri Tonu.
- Command: Codex / Düzelt.
- Command + Option: Codex / Chat.
- Command + Shift: Codex / Mail.
- Command + Control: Picker.

The extension also exposes `Help` in the provider dropdown and `Shortcut Help` inside Picker. These show a small macOS dialog with the shortcut list, then exit without rewriting text.

## Known Provider Findings

Local tests on the original development machine:

- Ollama local preset models worked and are the safest default for immediate feedback.
- Codex CLI worked and was the best practical provider for short rewrite tasks.
- OpenCode CLI worked but was slower and sometimes changed numeric punctuation, such as `2.04-2.27` to `2:04-2:27`. The prompt now explicitly asks providers to preserve numeric range/timecode punctuation.
- Gemini CLI was installed but failed with an account/client eligibility error at the time of testing.
- Claude CLI was installed but not logged in at the time of testing.

These findings are environment-specific. Other users may get different results depending on login state, subscription, CLI version, and default model.

## Prompt Requirements

The core behavior must remain strict:

- Return only the final rewritten text.
- Do not output Markdown, code fences, headings, options, analysis, or thinking text.
- Do not answer the selected text; rewrite it.
- Preserve meaning, speaker perspective, negation, uncertainty, names, links, dates, and commitments.
- Preserve punctuation style inside numeric ranges and timecodes exactly, for example `2.04-2.27` or `00:15-00:22`.

Turkish-specific risk cases:

- `çıksın` vs `çıkmasın`
- `çıkardık` vs `çıkarabiliriz`
- `2.04-2.27` as a timecode/range, not a date
- questions such as `var mı`, `olabilir mi`, `araştırır mısın` must remain questions, not be answered
- `ben/sen/biz` speaker perspective must not change

## Development Commands

Run smoke test:

```sh
tests/smoke.zsh
```

Run with explicit provider/preset:

```sh
env POPCLIP_OPTION_PROVIDER=codex POPCLIP_OPTION_PRESET=duzelt POPCLIP_OPTION_MODEL='' \
  ./LLM-CLI.popclipext/rewrite.zsh <<'EOF'
bu metni düzgün hale getirirmisin ama anlamı bozma
EOF
```

Validate JSON and shell syntax:

```sh
jq empty LLM-CLI.popclipext/Config.json
zsh -n LLM-CLI.popclipext/rewrite.zsh
zsh -n tests/smoke.zsh
```

Build zip:

```sh
rm -f dist/LLM-CLI.popclipext.zip
zip -r dist/LLM-CLI.popclipext.zip LLM-CLI.popclipext README.md LICENSE .gitignore tests CONTEXT.md
```

## Next Good Improvements

1. Add an optional picker workflow.
   - One PopClip icon.
   - Native macOS list via `osascript` or another default macOS mechanism.
   - User selects provider/preset/model combination at runtime.

2. Add preset configuration file support.
   - Example: `~/.config/popclip-llm-cli-rewrite/presets.json`
   - Lets power users define named provider/model/prompt combinations without editing the extension.

3. Add better timeout/error handling.
   - PopClip should show a useful error when a CLI is missing, not logged in, or takes too long.

4. Add provider-specific output cleanup.
   - OpenCode may emit plugin/status lines.
   - Codex uses `--output-last-message` to avoid noisy stdout.
   - Other providers may need similar handling.

5. Keep the public extension simple.
   - PopClip's official extension directory is curated.
   - The official PR may be rejected because the extension depends on external CLI tools.
   - The GitHub repo should remain usable regardless of official PopClip directory status.

## Guidance For Future Agents

If continuing this project with Claude, Codex, or another coding agent:

- Read `README.md`, then this file, then `LLM-CLI.popclipext/rewrite.zsh`.
- Do not assume PopClip supports native submenus unless new official docs prove it.
- Keep the extension source-only and inspectable.
- Avoid storing API keys or secrets.
- Prefer default macOS tools for picker UX if adding one.
- Keep the one-click/default behavior simple for public users.
- Treat provider-specific CLI behavior as unstable and test each provider separately.
